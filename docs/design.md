# Roundtable Novelbox Design Direction

## Product Position

中文名：圆桌小说盒子
英文名：Roundtable Novelbox

它不是通用聊天工具的复刻，而是面向长篇小说创作的个人 AI 共创器。它保留对话式工具“打开就能写”的直接感，但把核心差异放在小说长期记忆和多助手圆桌讨论上。

## Core Sentence

解决普通 AI 聊天难以持续管理长篇小说的问题，将对话创作、世界观设定、剧情线和正文库统一在一个移动端写作工作台中，并探索多助手圆桌共创模式。

## New Direction: Roundtable Co-Creation

圆桌共创模式把写作从“我问 AI 答”推进到“围绕同一份正文开会”：

- 正文缩成纸张，放在中上方，成为所有讨论的共同对象。
- 用户在下方和多个助手自然交流。
- 助手拥有不同立场：设定、剧情、审稿、文风、角色代入等。
- 被选中的助手按顺序阅读上文并发言。
- 助手可以赞同、补充，也可以反驳。
- 用户决定采纳、重写，或 `@写手` 让写手把讨论变成正文。
- 写手输出以正文框呈现，不混入普通聊天气泡。

## UI Principles

- 移动端优先。
- 第一屏必须是可用的创作界面，不做营销页。
- 正文纸张要成为圆桌模式最明显的视觉中心。
- 成员选择藏在菜单里，不在主界面铺满“参与席”。
- 标注 Beta 时保持诚实：展示方向，不夸大为完整多助手已完成。
- 视觉应像写作工具，不像通用客服聊天后台。

## Current Beta Entry

正式前端已加入“圆桌共创 Beta”入口，用于展示新方向：

- 顶部“圆”按钮。
- 纸张式正文小窗。
- 圆桌成员示例。
- 讨论预览。
- 明确提示完整多助手顺序发言仍在开发中。

## Roadmap

- 成员配置：新增、编辑、排序、勾选。
- 每个助手独立提示词、模型、温度。
- 按顺序触发助手发言。
- 模型上下文明确标注发言者，避免串消息。
- 深度分析对用户可见部分保持中文。
- `@写手` 将讨论总结为正文续写。
- 参与者审批、提意见、重写流。
