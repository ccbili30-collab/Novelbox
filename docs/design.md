# TBird Roundtable Box Design Direction

## Product Position

中文名：TBird 圆桌盒子
英文名：TBird Roundtable Box

TBird 圆桌盒子是下一代 AI 交流与创作工作台。它的核心不再是“一个 AI 帮我写小说”，而是“我把不同 AI 拉进同一场会议，让它们用不同偏好、气质和观点一起参与交流，最后把对话沉淀成作品”。

小说创作是第一个强场景，但不是产品边界。产品应支持用户把灵感、情绪、观点、草稿、矛盾、问题和目标带进圆桌，再由不同 AI 共同讨论、反驳、总结、起草、整理。

## Core Sentence

把灵感、情绪和观点，和不同 AI、不同声音、不同灵魂一起聊成作品。

## Roundtable Philosophy

- AI 进入圆桌时，首先是被拉进来开会的参与者，而不是被固定死的工具模块。
- 角色 prompt 只表达偏好，不表达牢笼。它可以让某个 AI 更偏结构、更偏情绪、更偏批评、更偏表达，但不应强制它只能做某一类事。
- 写手是执笔人，不是永远写小说的机器。需要小说正文时写正文；普通讨论时可以总结观点、写文章、整理会议结论或提出表达建议。
- 用户始终是主持人和最终决策者。AI 可以争论和建议，但不能替用户决定作品方向。
- 圆桌的主题由上下文决定。小说、产品、设计、人生问题、文章、演讲、世界观或日记都可以成为会议主题。

## Current UI Direction

- 移动端优先。
- 第一屏必须是可用的交流界面，不做营销页。
- 圆桌模式下，草稿 / 正文像一张纸放在讨论上方。
- 参会人设置贴着底部输入框打开，不固定在会话中间。
- 输入区负责主持会议：发言、`@`、开始本轮、停止本轮。
- 圆桌成员自然发言，不强制格式。
- `@` 像群聊一样拉起成员选择，支持插队和入队。
- 流式输出必须低闪烁：只更新当前消息文本节点，不整页刷新，不自动滚动，不闪光标。

## Prompt Direction

旧方向偏向“小说模块”：

- 世界观塑造者只能管世界观。
- 事件管理只能管事件。
- 角色管理只能管角色。
- 伏笔管理只能管伏笔。
- 写手只能写正文。

新方向应改成“人格偏好”：

- 世界观塑造者：偏好规则、系统、背景、边界和长期结构。
- 事件管理：偏好因果、推进、冲突、行动后果和节奏。
- 角色管理：偏好人物动机、关系、心理、欲望和可信度。
- 伏笔管理：偏好信息差、悬念、回收、误导和读者预期。
- 写手：偏好表达、总结、文章、段落和正文输出。

这些偏好不应阻止 AI 自由交流。系统只应告诉 AI：你被拉进一场圆桌会议，请根据当前主题、你的偏好和已有发言自然参与。

## Near-Term Roadmap

- 将默认成员提示词从“职责限制型”改为“偏好表达型”。
- 将写手改为“执笔人”：聊天时总结 / 写文章 / 整理观点，明确写小说时才写正文。
- 将小说资料面板逐步抽象成“创作材料 / 会议资料”。
- 保留小说字段作为强场景，但文案上不再把产品限制为小说工具。
- 完善低闪烁流式输出和圆桌消息增量更新。
- 推动 GitHub 展示、README、设计文档统一到 TBird 圆桌盒子的新定位。
