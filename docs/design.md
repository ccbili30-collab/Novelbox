# Roundtable Novelbox Design Direction

## Product Position

中文名：圆桌小说盒子
英文名：Roundtable Novelbox

它不是通用聊天工具的复刻，而是面向长篇小说创作的个人 AI 共创器。它保留对话式工具“打开就能写”的直接感，但把核心差异放在小说长期记忆和多助手圆桌讨论上。

## Visual Direction

当前 UI 分支采用的方向是：

`Cal` 的干净骨架 + `Claude` 的温暖纸感 + `Linear` 的秩序约束

翻译成产品语言，就是：

- 像一张整理好的写作桌，而不是一套花哨的聊天皮肤
- 稿纸是主角，但不做旧，不做复古装饰堆叠
- 讨论区要像安静会审，不像客服后台，也不像社交 App 的高饱和聊天页
- 行动按钮少、轻、明确，减少视觉噪音

## Clean Writing Desk Rules

- 背景以纸白、雾白、浅灰为主，不使用大面积高饱和色块
- 主操作色保持近黑或深墨色，强调“开始写”和“继续推进”
- 正文使用更适合长阅读的纸面字体气质，普通 UI 保持清楚、克制
- 圆桌模式里，稿纸必须是视觉中心，讨论区只做安静承托
- 讨论气泡按文字内容收缩，保持轻、薄、贴合，不做撑满整列的厚重聊天框
- 头像、徽标、角色色都应该低饱和，避免抢正文注意力
- 边界使用细线和轻阴影，不做厚重卡片堆叠
- 动效服务于展开、收起、切换、生成，不做装饰性炫动

## Core Sentence

解决普通 AI 聊天难以持续管理长篇小说的问题，将对话创作、世界观设定、剧情线和正文库统一在一个移动端写作工作台中，并探索多助手圆桌共创模式。

## New Direction: Roundtable Co-Creation

圆桌共创模式把写作从“我问 AI 答”推进到“围绕同一份正文开会”：

- 正文缩成纸张，放在中上方，成为所有讨论的共同对象。
- 用户在下方和多个助手自然交流。
- 助手拥有不同立场：设定、剧情、审稿、文风、角色代入等。
- 被选中的助手按顺序阅读上文并发言。
- 助手可以赞同、补充，也可以反驳。
- 用户决定采纳、重写，或 `@写手` 让写手把讨论变成正文。
- 写手输出以正文框呈现，不混入普通聊天气泡。

## UI Principles

- 移动端优先。
- 第一屏必须是可用的创作界面，不做营销页。
- 正文纸张要成为圆桌模式最明显的视觉中心。
- 成员选择藏在菜单里，不在主界面铺满“参与席”。
- 标注 Beta 时保持诚实：展示方向，不夸大为完整多助手已完成。
- 视觉应像写作工具，不像通用客服聊天后台。
- 所有优化优先服务“更想写下去”，而不是“更像某个 AI 产品”。

## Current Beta Entry

正式前端已加入“圆桌共创 Beta”入口，用于展示新方向：

- 顶部“圆”按钮。
- 纸张式正文小窗。
- 圆桌成员示例。
- 讨论预览。
- 明确提示完整多助手顺序发言仍在开发中。

## Roadmap

- 成员配置：新增、编辑、排序、勾选。
- 每个助手独立提示词、模型、温度。
- 按顺序触发助手发言。
- 模型上下文明确标注发言者，避免串消息。
- 深度分析对用户可见部分保持中文。
- `@写手` 将讨论总结为正文续写。
- 参与者审批、提意见、重写流。
