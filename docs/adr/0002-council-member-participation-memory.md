# ADR 0002：议员参会记忆与主创状态

日期：2026-05-12

## 状态

提议中

## 背景

圆桌方向进一步明确：产品仍然使用“议员”这个成员称呼，但议员不只是固定功能模块。每个议员都应该像一个有经历的独立参与者，可以拥有自己的私聊会话、偏好、历史和参会记录。

用户希望圆桌像一群有不同经历的主创来到同一个群里交流。议员在某场圆桌里可以承担主创职责，但主创不是永久身份。写手仍然是输出通道，负责续写、总结、整理成稿，不承担主要思想主体。

## 决策

保留“议员”作为长期成员模型。

增加“主创状态”作为议员在某个圆桌话题里的临时参与状态。

后续需要设计独立的议员参会记录层：

- 记录议员参加过哪些圆桌。
- 记录议员在群聊里说过什么。
- 记录与该议员相关的群聊摘录和判断。
- 这些记录可以被议员自己的私聊会话查询。
- 这些记录默认不直接占用主会话上下文。

## 影响

这会避免把所有上下文都塞进当前创作会话，降低主会话膨胀。

议员可以形成更稳定的长期人格和判断，不会因为离开圆桌或回到私聊就“失忆”。

写手继续保持工具化：它主要负责把用户和议员讨论出来的内容变成正文、总结、文章或方案。

## 后续工作

- 设计议员私聊会话和圆桌参会记录的数据结构。
- 设计“拉人进入圆桌”的交互。
- 设计议员回到私聊时如何查询参会记录。
- 让 AI 上下文构建器可以按需读取议员参会记录，而不是默认塞入主会话。
