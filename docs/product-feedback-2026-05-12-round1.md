# 第一轮产品批注

日期：2026-05-12

来源：`C:\Users\16014\Desktop\untitled.md`

## 核心方向

后续重构和 UI 优化必须服务“干净、整洁、排除杂念、行动丝滑”的移动端创作体验。

圆桌模式不是普通聊天的复制。圆桌正文栏应严格围绕写手输出分段，进入圆桌后写手更像受约束的正文输出者，不再像交流模式一样随意聊天。

## 关键约束

1. 历史会话的新建按钮不能使用看不清的白色。
2. 历史会话需要重命名功能。
3. 交流模式点击气泡不能触发明显刷新闪动。
4. 左下角设置入口应改为写手入口；三点菜单负责美术调整和默认 API/模型提供方。
5. 圆桌模式写手按钮应放到写手画布顶部，可与“主线对话摘录”一起收起。
6. 画布展开条继续缩小，只保留贴合文字的空间。
7. 写手和议员设置都需要联网开关。
8. 上下文预览移到输入框上方，透明背景；原位置改为 `+`，用于图片等输入。
9. 议员设置界面整体缩字、缩按钮，去掉模板套用，取消按钮改为右上角固定关闭。
10. 议员导入导出要改为 TBird 自己的人格文本格式，并支持参会人批量导入导出。
11. 圆桌起步介绍“把正文放到桌上”应去掉，不要教用户做事。
12. 圆桌讨论应直接位于底层背景，正文摘录是覆盖在上层的纸张，不影响下层滑动。
13. 顶层 UI 继续压缩，尽量把中间留给创作。
14. 本地背景上传不应覆盖米黄色蒙版。
15. 可以考虑双指捏合进入圆桌模式，双击正文栏回交流模式。
16. 排版调试改为数值输入，并支持展开/收起。
17. 三点菜单新增“模型提供方”：用户填 API、配置模型；后续写手/议员不直接填 API，而是从提供方选择模型。
18. 交流模式“发到圆桌”功能意义较弱，后续应重新评估或移除。
19. 正文栏需要按写手输出卡片状分段，圆桌正文应来自写手输出同步。
20. 议员输出不应重复自报名字，例如“世界观塑造者：世界观塑造者：……”。
21. 所有弹窗应支持点外部关闭和手机返回手势关闭。
22. 圆桌模式默认展开正文栏 10%。

## 对架构重构的影响

- 不继续强化“每个议员都有独立 API 表单”的方向；保留兼容层即可。
- 优先抽写手正文同步、写手输出分段、模型提供方、弹窗关闭行为。
- 圆桌相关模型应区分“讨论消息”和“写手正文片段”。
- UI 优化要优先减少重渲染、减少按钮重复、压缩顶部控件和设置面板。
