# TBird 圆桌盒子 / TBird Roundtable Box

**下一代 AI 交流与创作工作台。**

一句话：**把灵感、情绪和观点，和不同 AI、不同声音、不同灵魂一起聊成作品。**

在线展示：[https://ccbili30-collab.github.io/Novelbox/](https://ccbili30-collab.github.io/Novelbox/)

## 新方向

TBird 圆桌盒子不再只是一个类似 Chatbox 的单助手小说创作工具。

它正在转向一个移动端优先的 AI 圆桌工作台：用户可以把多个 AI 拉进同一场对话，让它们按顺序发言、互相回应、通过 `@` 插队或入队，并把零散想法逐步聊成可以保留的作品。

小说仍然是第一个强场景，但不再是产品边界。圆桌可以讨论小说、产品想法、个人问题、设计方向、演讲稿、文章、世界观、情绪困境或任何需要多人观点碰撞的主题。最后沉淀出来的东西可以是正文、总结、文章、方案、设定集、决策记录或新的创作方向。

核心不是“我问一个 AI，它回答我”，而是：

> 我把几个 AI 带进一个房间，让它们一起思考，然后由我决定什么会成为作品。

## 关键差异

- **群聊式交流，而不是单线聊天。** 参与者可以被选择、排序、点名、拉入下一轮。
- **角色提示词是偏好，不是牢笼。** AI 可以偏好结构、情绪、批评、世界观、写作或审稿，但系统不该把它强行限制成某个工具按钮。
- **写手是灵活的执笔人。** 明确写小说时它写正文；普通讨论时它可以总结观点、组织语言、写文章或整理会议结论。
- **用户保留决定权。** AI 可以争论、建议、总结、起草；采纳权始终在用户手里。
- **移动端优先。** 适合手机上快速捕捉、长时间交流和专注创作。
- **创作记忆。** 正文、设定、角色、剧情、大纲、伏笔等材料仍可作为会议资料进入上下文。

## 当前功能

### 对话工作台

- 移动端优先的聊天界面。
- 历史会话、新建会话、复制会话、删除会话。
- 用户消息编辑。
- AI 输出真实编辑。
- AI 回复版本切换。
- OpenAI 兼容 API 配置。
- 拉取模型列表。
- 温度、上下文条数、无限上下文、最大输出 token。
- 停止生成。
- 低闪烁流式输出：流式片段会合并后只更新当前消息节点，不整屏刷新。

### 圆桌模式

- 普通对话和圆桌模式一键切换。
- 正文 / 草稿纸片放在讨论区上方。
- 纸片支持拖动高度和折叠。
- 参会人设置面板贴着底部输入框打开。
- 通过数字选择并排序参会人。
- 点击开始本轮后，选中的参会人按顺序发言。
- `@` 选择浮窗，用于把已选择的参会人拉进后续发言队列。
- 多个 `@` 可以按顺序入队。
- 每个参会人可单独设置提示词、模型、API Base URL、API Key、温度、最大输出和可见材料范围。
- 失败隔离：某个参会人 API 失败不应拖死整轮讨论。
- 激活层：被激活的参会人可以更自然地理解会议动态、争论、离席和关系变化。

### 创作记忆

项目仍然保留长内容创作所需的资料面板：

- 正文库。
- 剧情线。
- 角色卡。
- 世界观。
- 大纲。
- 伏笔线。
- TXT 导入和导出。
- AI 自动整理资料。
- 上下文预览和 token 估算。

这些材料现在应被理解为**可选会议资料**，而不是产品只能做小说的证明。

### Android

仓库包含 Android WebView 包装工程：

```text
android-app/
```

构建 debug APK：

```powershell
cd android-app
.\gradlew.bat assembleDebug --offline --no-daemon
```

APK 输出：

```text
android-app/app/build/outputs/apk/debug/app-debug.apk
```

## 本地 Web 运行

```bash
node dev-server.mjs
```

默认地址：

```text
http://127.0.0.1:5177/
```

## 产品宣言

**TBird 圆桌盒子是一个 AI 圆桌：把内在的材料变成外在的作品，包括灵感、情绪、观点、草稿、争论和决定。**

它不只是聊天壳，而是一个和不同 AI 一起思考，并让交流沉淀为作品的地方。
