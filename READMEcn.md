# 圆桌小说盒子 / Roundtable Novelbox

一个移动端优先的 AI 小说创作工作台。

一句话：解决普通 AI 聊天难以持续管理长篇小说的问题，将对话创作、世界观设定、剧情线和正文库统一在一个移动端写作工作台中，并探索多助手圆桌共创模式。

在线展示：[https://ccbili30-collab.github.io/Novelbox/](https://ccbili30-collab.github.io/Novelbox/)

## 灵感来源

圆桌小说盒子的交互灵感来自对话式 AI 工具：打开就能写，随时能追问、修改、续写，不需要先进入一套复杂工作台。

但它不想停留在“聊天工具”。长篇小说创作真正麻烦的地方，是正文越写越长、设定越来越散、角色状态不断漂移、AI 不知道自己到底该看哪一段资料。圆桌小说盒子把这些问题收回到一个移动端写作场景里：聊天负责即时创作，正文库和资料面板负责长期记忆，圆桌共创负责多人视角的讨论和审稿。

## 隆重介绍：圆桌共创方向

圆桌共创是本项目的新方向，也是它和普通 AI 聊天工具拉开距离的地方。

设想中的工作流是这样的：

- 默认情况下，用户和“写手”一对一创作，像正常聊天一样推进小说。
- 需要讨论时，用户打开圆桌模式，把正文缩成一张放在中上方的纸。
- 下方变成会议式讨论区，设定师、剧情师、审稿、文风师等助手按用户选择的顺序发言。
- 每个助手都知道“谁说了什么”，能阅读前面的讨论，也可以提出不同意见。
- 用户最终决定采纳、重写，或 `@写手` 把讨论转回正文续写。
- 写手输出不是普通气泡，而是正文框，天然和普通讨论区分开。

当前版本已经在前端放入“圆桌共创 Beta”入口，用于展示这个产品方向。完整的多助手顺序发言、成员配置和独立提示词系统正在开发中。

## 已完成能力

### 1. 移动端对话创作

- 对话优先，第一屏就是写作。
- 用户消息靠右，AI 消息靠左。
- 底部输入栏固定，适合手机操作。
- 支持历史会话、新建、复制、删除。
- 支持 AI 回复版本切换。
- 支持编辑用户内容。
- 支持直接编辑 AI 输出。

这里的“编辑 AI 输出”不是把修改内容当成新 prompt 再发一次，而是直接改当前 assistant 消息内容。后续上下文读取的就是你改过的版本。

### 2. OpenAI-compatible API

- 支持 Base URL。
- 支持 API Key。
- 支持模型名输入。
- 支持拉取模型列表。
- 支持温度、上下文条数、无限上下文、最大输出 token。
- 支持流式输出。
- 支持停止生成。

没有本地模拟糊弄人。没填 API Key 就明确报错，不会偷偷生成假内容。

### 3. 小说资料系统

圆桌小说盒子有一个独立的“小说资料”面板：

- 正文库
- 剧情线
- 角色卡
- 世界观
- 大纲
- 伏笔线

这些不是装饰。它们会进入“本次上下文预览”，也会进入真实 API 请求，帮助 AI 在长篇创作里记得住东西。

### 4. 正文导入与导出

- 支持 TXT 正文导入。
- 支持正文导出为 TXT。
- 正文库会作为小说记忆材料进入上下文。
- 正文库默认取尾部节选进入上下文，避免一次把模型塞爆。

### 5. AI 自动归纳和资料填充

每个资料区都可以让 AI 自动整理：

- AI 归纳剧情线
- AI 填充角色卡
- AI 填充世界观
- AI 填充大纲
- AI 填充伏笔线

这些功能走真实 API。不会用代码截取冒充“AI 生成”。

### 6. 上下文预览

你可以查看本次实际发送给 AI 的上下文：

- 系统提示
- 小说资料记忆
- 最近对话
- 当前输入
- 估算 token
- 模型和温度

长篇创作里，最大的痛苦不是 AI 写得差，而是你根本不知道它到底看没看你给的正文和设定。上下文预览就是为了解决这个问题。

### 7. 排版调试面板

为了让移动端写作真的舒服，内置排版调试：

- 输入栏高度
- 输入框字号
- 发送按钮大小
- 工具按钮大小
- 消息字号
- 行间距
- AI 左边距
- 消息两侧边距
- 消息间距
- 用户气泡内边距
- 元信息字号
- 省略号大小

还可以：

- 使用经典紧凑预设
- 使用大字舒适预设
- 保存自己的排版预设
- 删除预设
- 复制排版参数
- 恢复默认

## 安卓无负担使用

项目带有 Android WebView 包装工程：

```text
android-app/
```

构建 debug APK：

```bash
cd android-app
./gradlew assembleDebug
```

Windows：

```powershell
cd android-app
.\gradlew.bat assembleDebug
```

构建产物：

```text
android-app/app/build/outputs/apk/debug/app-debug.apk
```

装上 APK，填 API Key 和模型，就可以直接使用。

## Web 本地运行

```bash
node dev-server.mjs
```

默认地址：

```text
http://127.0.0.1:5177/
```
