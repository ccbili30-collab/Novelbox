# Novelbox / TBird

一个移动端优先的 AI 小说创作盒子。

它的目标很朴素：像 Chatbox 一样顺手聊天，但比普通聊天工具更懂小说创作。能在安卓手机上轻松跑，能接 OpenAI-compatible API，能保存小说资料，能把正文、剧情线、角色卡、世界观这些东西真正送进上下文，而不是在界面上摆着好看。

## 作者的怨念

这个项目的诞生，带着作者对 Chatbox 的复杂感情。

Chatbox 好用，好就好在它是聊天，不是工作台；它让人可以随便问、随便改、随便续。但一旦开始拿它写长篇小说，问题就会一个接一个冒出来：正文太长，AI 忘设定；剧情线散了，人物开始失忆；用户想改 AI 输出，却发现很多工具只是“重新生成”，不是实打实地改历史消息。更糟糕的是，修 UI 的过程也很容易变成痛苦修代码：这里补一个按钮，那里补一个弹窗，最后变成一座谁也不想爬的代码山。

Novelbox 就是在这种怨念里重做出来的。目标不是炫技，而是把小说写作这件事重新压回一个舒服的移动端聊天体验里。

## 隆重介绍：我们到底优化了什么

### 1. Chatbox 式移动端聊天

- 对话优先，第一屏就是聊天。
- 用户消息靠右，AI 消息靠左。
- 底部输入栏固定，不搞复杂工作台。
- 支持历史会话、新建、复制、删除。
- 支持 AI 回复版本切换。
- 支持编辑用户内容。
- 支持直接编辑 AI 输出。

这里的“编辑 AI 输出”不是把修改内容当成新 prompt 再发一次，而是直接改当前 assistant 消息内容。后续上下文读取的就是你改过的版本。

### 2. OpenAI-compatible API

- 支持 Base URL。
- 支持 API Key。
- 支持模型名输入。
- 支持拉取模型列表。
- 支持温度、上下文条数、无限上下文、最大输出 token。
- 支持流式输出。
- 支持停止生成。

没有本地模拟糊弄人。没填 API Key 就明确报错，不会偷偷生成假内容。

### 3. 小说资料系统

Novelbox 现在有一个独立的“小说资料”面板：

- 正文库
- 剧情线
- 角色卡
- 世界观
- 大纲
- 伏笔线

这些不是装饰。它们会进入“本次上下文预览”，也会进入真实 API 请求，帮助 AI 在长篇创作里记得住东西。

### 4. 正文导入与导出

- 支持 TXT 正文导入。
- 支持正文导出为 TXT。
- 正文库会作为小说记忆材料进入上下文。
- 正文库默认取尾部节选进入上下文，避免一次把模型塞爆。

### 5. AI 自动归纳和资料填充

每个资料区都可以让 AI 自动整理：

- AI 归纳剧情线
- AI 填充角色卡
- AI 填充世界观
- AI 填充大纲
- AI 填充伏笔线

这些功能走真实 API。不会用代码截取冒充“AI 生成”。

### 6. 上下文预览

你可以查看本次实际发送给 AI 的上下文：

- 系统提示
- 小说资料记忆
- 最近对话
- 当前输入
- 估算 token
- 模型和温度

这点非常重要。因为长篇小说创作里，最大的痛苦不是 AI 写得差，而是你根本不知道它到底看没看你给的正文和设定。

### 7. 排版调试面板

为了避免“截图给人改 CSS，改十次还不舒服”的折磨，Novelbox 内置排版调试：

- 输入栏高度
- 输入框字号
- 发送按钮大小
- 工具按钮大小
- 消息字号
- 行间距
- AI 左边距
- 消息两侧边距
- 消息间距
- 用户气泡内边距
- 元信息字号
- 省略号大小

还可以：

- 使用 Chatbox 紧凑预设
- 使用大字舒适预设
- 保存自己的排版预设
- 删除预设
- 复制排版参数
- 恢复默认

你可以在手机上自己拖滑块调到顺眼，不必每次都回来修代码。

## 安卓无负担使用

Novelbox 已经带了一个 Android WebView 包装工程：

```text
android-app/
```

构建 debug APK：

```bash
cd android-app
./gradlew assembleDebug
```

Windows：

```powershell
cd android-app
.\gradlew.bat assembleDebug
```

构建产物：

```text
android-app/app/build/outputs/apk/debug/app-debug.apk
```

它本质是轻量 WebView，不需要你在手机上折腾复杂环境。装上 APK，填 API Key 和模型，就可以直接使用。

## Web 本地运行

项目也可以作为本地 Web 原型运行：

```bash
node dev-server.mjs
```

默认地址：

```text
http://127.0.0.1:5177/
```
